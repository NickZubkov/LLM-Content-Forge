# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Состояние проекта

Пустой (greenfield) проект Unity, созданный по шаблону **2D (URP)**. На данный момент в `Assets/` **нет C#-кода** — только сцена-шаблон, ассеты настроек и файл Input System. Архитектурные решения ещё не приняты; подстраиваться пока не под что.

- **Движок:** Unity `6000.3.19f1` (Unity 6.3). Версия редактора зафиксирована в `ProjectSettings/ProjectVersion.txt` — используй ровно её; открытие другой версией молча переимпортирует и перезапишет ассеты.
- **Render pipeline:** Universal RP 17.3.0, настроен под 2D (`Assets/Settings/Renderer2D.asset`, `UniversalRP.asset`).
- **Ввод:** новый Input System (1.19.0). Действия игрока лежат в `Assets/InputSystem_Actions.inputactions` — редактируй их, а не добавляй устаревший опрос через `Input.GetKey`.

## Работа с репозиторием

В репозиторий не закоммичена CLI-точка входа для сборки/тестов. Все сборки, запуски play-mode и тесты идут через редактор Unity (либо через `-batchmode` CLI на зафиксированной версии редактора). Важное:

- **Тесты:** `com.unity.test-framework` (1.6.0) установлен, но тестов пока нет. Добавляя тесты, создай assembly definition (`.asmdef`) для тестовой сборки со ссылками на NUnit и тестируемый код; запускай через **Window > General > Test Runner** (вкладки EditMode / PlayMode). Соглашения о папке `Tests/` ещё нет — задай его.
- **Новый код:** добавляя первые скрипты, заводи `.asmdef`-файлы, чтобы держать время компиляции и границы тестов под контролем, а не сваливать всё в дефолтную `Assembly-CSharp`.
- **`.sln` / `.csproj`:** генерируются пакетами интеграции с IDE (Rider / Visual Studio), а не пишутся вручную. Не коммить и не правь их руками — они пересоздаются при открытии проекта.

## Автоматизация редактора Unity через MCP

Доступен скилл `unity-mcp-skill` для управления редактором (GameObject'ы, скрипты, сцены, тесты) по MCP. Предпочитай его для операций на стороне редактора вместо ручной правки YAML в `.unity` / `.asset` — она хрупкая, ассеты ссылаются друг на друга по GUID.

## Ассеты и meta-файлы

- У каждого ассета есть парный `.meta`-файл с GUID и настройками импорта. **Всегда перемещай/переименовывай/удаляй ассеты через редактор (или MCP)**, чтобы `.meta`-файлы и перекрёстные ссылки по GUID оставались согласованными. Прямая правка YAML по пути осиротит ссылки.
- `Library/`, `Temp/`, `Logs/`, `obj/` и сгенерированные файлы проекта IDE игнорируются через `.gitignore`; `.gitattributes` настраивает Git LFS и правила merge для YAML — сохраняй сериализацию ассетов в режиме **Force Text** (дефолт проекта), иначе diff и merge сломаются.
